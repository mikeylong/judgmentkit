# JudgmentKit 0.2.0

Released: 2026-06-12

## Accessibility Contracts

JudgmentKit 0.2.0 makes accessibility evidence a first-class part of the UI implementation contract and MCP review flow.

- Adds `implementation_contract.accessibility_policy` to the UI implementation contract.
- Defaults the policy to WCAG 2.2 AA, with contrast targets of `4.5:1` for normal text, `3:1` for large text, and `3:1` for non-text UI.
- Defines machine-readable contract groups for readability and contrast, keyboard and focus, semantics and structure, forms/status/errors, motion/media/timing, and responsive/input behavior.
- Requires core evidence for automated checks, semantic content, landmarks/headings, name/role/value, keyboard navigation, focus order, focus-visible, and responsive no-overflow.
- Adds conditional evidence expectations for visual-background contrast, non-text contrast, forced-colors, target size, focus-not-obscured, no keyboard trap, reduced motion, pause/stop/hide, hover/focus content, form labels/errors, status messages, media alternatives, and semantic fallbacks.
- Treats automated scans and screenshots as supporting artifacts, not replacements for browser-rendered contrast, keyboard, semantic, and responsive evidence.

## MCP Tool Behavior

- `create_ui_implementation_contract` accepts an `accessibility_policy` override and returns the normalized policy.
- `review_ui_implementation_candidate` validates structured `accessibility_evidence`, accepts `covered_states` and `static_evidence` aliases, fails reported accessibility failures, and rejects visual-background contrast evidence below target.
- `create_frontend_generation_context` carries the policy as `implementation_guidance.accessibility_policy`.
- `create_frontend_implementation_skill_context` compiles the policy into agent-facing instructions, guardrails, and verification checklist items without exposing MCP machinery in product UI guidance.

## Documentation And Demos

- Adds docs and skill guidance for visual-heavy, keyboard, semantic, form/status, motion, and responsive accessibility evidence.
- Adds a search-page accessibility comparison demo with unguided and JudgmentKit MCP-guided standalone HTML pages.
- Adds MCP planning cards and structured content that summarize accessibility evidence expectations for agents and reviewers.
