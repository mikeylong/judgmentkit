# JudgmentKit 0.5.0

Released: 2026-06-20

## Cognitive Dimensions Review

JudgmentKit 0.5.0 adds a Cognitive Dimensions review pass for LLM-generated UI workflow and implementation candidates.

- Adds `review_cognitive_dimensions_candidate` to the library, stdio MCP server, and hosted MCP metadata.
- Reviews whether a candidate maps to the user's domain work instead of source mechanics.
- Checks whether decision, evidence, dependencies, commitment boundaries, progressive evaluation, correction cost, memory-heavy transitions, role clarity, and disclosure discipline are strong enough before UI generation.
- Keeps Cognitive Dimensions language diagnostic: agents should use findings to repair the candidate, not copy the terminology into product UI.

## Handoff Gate

The UI generation handoff can now accept a Cognitive Dimensions review packet.

- A supplied Cognitive Dimensions review must be `ready_for_review`.
- Non-ready reviews block handoff with structured blocker details.
- Ready reviews are recorded as an additional generation gate, alongside workflow and implementation contract gates.

## Design-System and Site QA

This release also hardens the public design-system reference that agents and humans use to review generated UI.

- Preserves system-detected light/dark mode with no visible appearance toggle by default.
- Fixes dark-mode contrast and usability issues across navigation menus, report surfaces, system-map rendering, examples gallery metadata, and design-system step markers.
- Adds regression checks for dark-mode token surfaces, menu semantics, current-page state, modal focus containment, React Flow map theming, and numbered marker contrast.
- Keeps the design-system site human-first while exposing deterministic semantic exports for agents.

## Evidence

- Expanded MCP pilot cases with Cognitive Dimensions scenarios.
- Added cognitive-dimensions capture and report artifacts.
- Refreshed model UI reviewed-handoff fixtures for the release contract version.
- Added ED flow board example coverage.
