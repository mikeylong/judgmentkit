MCP 0.4.0 review artifact

Portable design assets in JudgmentKit MCP

JudgmentKit now sends token roles, system font stacks, and Lucide icon catalog policy through the existing implementation contract and frontend implementation skill context. The full generated Lucide catalog ships with MCP catalog tools and remains adapter-layer guidance, not a substitute for activity, workflow, disclosure, accessibility, static-check, or browser-QA gates.

Tokens

Semantic token roles are normalized inside implementation_contract.visual_token_adapter and exposed in MCP planning cards.

surface text border focus status decision risk receipt

Fonts

No CDN or bundled font files are required. Defaults use portable CSS system stacks.

  • body: system-ui stack
  • heading: system-ui stack
  • label: system-ui stack
  • numeric: system stack with tabular numbers
  • diagnostic: ui-monospace stack

Icons

No runtime icon CDN or remote fetch is required. MCP structured content includes the Lucide catalog summary, source/version, count, policy, and tool names; SVG payloads are retrieved only through icon catalog tools.

  • Committed generated catalog: Lucide 1.21.0, 1,737 canonical icons.
  • Use search_icon_catalog and get_icon_svg to choose and retrieve concrete icons.
  • Icon-only controls still need accessible names and non-text contrast evidence.

Verified paths

  • create_ui_implementation_contract exposes token, font, and icon defaults.
  • create_frontend_implementation_skill_context includes normalized token/font/icon guidance.
  • MCP stdio and library tests assert system font stacks plus Lucide catalog list/search/get tool behavior.

Still guarded

  • Unsupported font and icon roles fail visual-token evidence review.
  • Design assets cannot satisfy missing workflow, primitive, state, action-boundary, data-visibility, accessibility, static-check, or browser-QA gates.
  • Material UI remains an override/example adapter, not the default renderer.
